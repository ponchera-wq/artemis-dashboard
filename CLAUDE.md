# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A static, single-page PWA dashboard for tracking the NASA Artemis II crewed lunar flyby mission in real time. No build step, no framework, no package manager — open `index.html` directly in a browser or serve it locally.

## Running Locally

```bash
# Any static file server works; python is usually available:
python3 -m http.server 8080
# then open http://localhost:8080
```

There are no tests, no linter config, and no CI pipeline.

## Architecture

All code is vanilla JS (ES5/ES6 mixed, no modules, no bundler). Every JS file runs as an IIFE and communicates via globals on `window`.

### Shared globals (`js/shared.js` — loaded first)

| Global | Purpose |
|---|---|
| `window.LAUNCH_UTC` | Mission epoch (`Date` object). All elapsed-time math anchors here. |
| `window.dashboardState` | Cross-module state: `useImperial`, `usingLive`, `currentPhase`, `elapsed` |
| `window.fmtLocal` / `window.fmtUTC` / `window.tzAbbr` | Time formatting helpers |
| `window.NASA_API_KEY` | Public demo key for NASA APIs |

### Shared data modules (loaded after shared.js, before consumers)

| Global | Source file | Purpose |
|---|---|---|
| `window.MissionEphemeris` | `js/mission-ephemeris.js` | Single source of truth for spacecraft & Moon state. Loads `data/mission-ephemeris.json` (644 pre-computed points from OEM + skyfield). `getState(metSec)` returns interpolated position, velocity, distances, speed. |
| `window.MissionEvents` | `js/mission-events.js` | Canonical event list (59 events). Merges trajectory waypoints and timeline events. Provides `getActiveEvent()`, `getNextEvent()`, `getWaypoints()`, `getEventsByPhase()`. |

### Ephemeris data (`data/mission-ephemeris.json`)

Generated by `data/generate_ephemeris.py` from the NASA/JSC/FOD/FDO OEM file (`artemis2_oem.asc`) + Moon positions via skyfield/DE440s. All coordinates in EME2000 (km, km/s). Data covers MET ~16,370s to ~782,292s (~T+4.5h to T+217.3h). The generator is a build-time-only dependency (`pip install skyfield`).

Each point contains: `epochUtc`, `metSec`, `orion:{x,y,z,vx,vy,vz}`, `moon:{x,y,z}`, `distEarthKm`, `distMoonKm`, `speedKms`.

**Note:** The OEM trajectory revision places closest approach at ~T+120.4h (MET 433,479s), not the press kit's T+128h. Event timings in mission-events.js reflect the OEM trajectory.

### Module responsibilities

| File | Responsibility |
|---|---|
| `js/clock.js` | MET countdown/elapsed clock, mission phase detection, signal pill |
| `js/stats.js` | Telemetry display — reads distances and speed from `MissionEphemeris.getState()`. Shows 'OEM' badge when in data range, 'EST' when outside. |
| `js/trajectory.js` | Three.js 3D scene: Earth, Moon (dynamic position from ephemeris), spacecraft path, holographic HUD overlay canvas. Uses `MissionEphemeris` for all position/velocity data and `MissionEvents` for waypoint labels. |
| `js/timeline.js` | Mission event list with phase filtering, per-event ETAs. Reads from `MissionEvents.events`. |
| `js/dsn.js` | Polls `eyes.nasa.gov/dsn/data/dsn.xml` every 30s for Deep Space Network antenna data |
| `js/weather.js` | Polls NOAA JSON endpoints for Kp index (geomagnetic storm), solar wind, X-ray flux gauges |
| `js/news.js` | Fetches NASA/Space.com/etc. RSS via `api.rss2json.com`, tabs by source |
| `js/crew.js` | Static crew profiles with modal detail cards |
| `js/ui.js` | YouTube iframe tabs, telemetry cinematic overlay, drag-to-reorder panels, unit toggle |

### External libraries (CDN, no local copies)

- **Three.js r128** — 3D trajectory scene

### External APIs

- **NASA APOD** (`api.nasa.gov`) — key stored in `window.NASA_API_KEY` (public demo key, rate-limited)
- **DSN Now** (`eyes.nasa.gov/dsn/data/dsn.xml`) — antenna telemetry
- **NOAA SWPC** — space weather JSON endpoints
- **rss2json.com** — RSS-to-JSON proxy for news feeds

### Panel layout

Panels are draggable via `js/ui.js`. Order is stored in `localStorage` under `artemis-panel-order`.

### PWA

`manifest.json` + `service-worker.js` enable installability. The service worker caches the shell for offline use.
